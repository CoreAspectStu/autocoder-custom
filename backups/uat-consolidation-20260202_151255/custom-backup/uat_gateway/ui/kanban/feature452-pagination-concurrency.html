Why This Matters

In multi-user environments, it's common for one user to be viewing paginated results while another user adds new data. The pagination system must handle this gracefully:

  • Offset-based pagination (used here) is stable - page 2 always shows results [limit:limit*2]
  • New results are added to the end, so they don't disrupt pages currently being viewed
  • This prevents "shifting pages" where results move between pages as new data arrives
