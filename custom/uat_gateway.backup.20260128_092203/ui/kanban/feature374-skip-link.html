Feature #374: Skip to Content Link

WCAG 2.1 Compliance

This page demonstrates a properly implemented "Skip to Content" link that complies with WCAG 2.1 Success Criterion 2.4.1 - "Bypass Blocks".

What is a Skip Link?

A skip link (or "skip to content" link) is an accessibility feature that allows keyboard users to bypass repetitive navigation and jump directly to the main content of a page. This is especially helpful for:

  • Screen reader users who don't want to hear navigation on every page
  • Keyboard-only users who want to avoid tabbing through long navigation menus
  • Users with motor disabilities who find repetitive navigation tedious

Technical Implementation

HTML Structure:

<a class="skip-link" href="#main-content">
    Skip to main content
</a>

<nav>
    <!-- Navigation links -->
</nav>

<main id="main-content" tabindex="-1">
    <!-- Main content -->
</main>

CSS Styling:

.skip-link {
    position: absolute;
    top: -40px;  /* Hidden above viewport */
    left: 0;
    background: #1d4ed8;
    color: #ffffff;
    padding: 8px 16px;
    z-index: 10000;
}

.skip-link:focus {
    top: 0;  /* Visible when focused */
    outline: 3px solid #1e40af;
}

main:focus {
    outline: 3px solid #1d4ed8;  /* Focus indicator */
}

Key Implementation Details

‚úÖ Correct Implementation:

  • Hidden by default: top: -40px positions link above viewport
  • Visible on focus: :focus pseudoclass moves it to top: 0
  • High z-index: z-index: 10000 ensures it appears above all content
  • Smooth transition: transition: top 0.3s for smooth appearance
  • Target tabindex: main tabindex="-1" allows it to receive focus
  • Focus indicator: Clear outline on main when focused
  • High contrast: Blue background (#1d4ed8) with white text meets WCAG AAA
